## partitions.csv 写法

### 第一行一般都是`#Name,Type,Subtype,Offset,Size,Flags`

+  `Name`（名称） 就是起个绰号，可以是任意字符串，但**必须小于16个字符**。

+ `Type`（主类型） 决定了类型，常用的有两种：
    * `app` ： 应用程序，用来存放可以执行的代码。
    * `data` ：数据，用来存放配置、图片、文件等。

+ `Subtype`（子类型） 决定了具体的用途，选项取决于前面的 `Type` 的内容
    * **如果填了 `app`**
        * `factory` ：**出场固件**，一般情况下上电Bootloader会先找这个。
        * `test` ：**工厂测试固件**，用于量产测试。
        * `ota_0 ~ ota_15` : **OTA升级槽位**，做远程升级所需要的分区。

    * **如果填了 `data`**
        * `nvs` ：**键值对存储**，适合用来存放WiFi密码、配置结构体等，系统会在这个分区上做磨损均衡。
        * `phy` ：**射频参数**，用来存放WiFi/蓝牙的硬件校准数据。
        * `ota` ：**OTA引导数据**，用来记录当前所需要运行 `app` 的版本（对应上面的ota_0 ~ ota_15）。

+ `Offset` （偏移地址）用来说明这部分内存在Flash上的物理起始地址，可以选择**留空**，让编译器自己计算，也可以选择具体的**十六进制地址**，但必须是4KB对齐。

+ `Size` （大小）用来规定所占空间大小，可以填：
    * **十进制带单位**：1M，2M，4M（适合大分区）；4k，16k（适合小分区）。
    * **十六进制**：0x1000（4KB），0x4000（16KB）。
    * **必须是4KB（0x1000）的整数倍**，因为Flash擦除的最小单位Sector是4KB（一般对于外挂Flash来说）。

+ `Flags` （标志位）一般留空，也就是普通分区，填入`encrypted`就可以变成加密分区，可以防止别人读取固件或者密钥。
